# Audience Analyzer — CLAUDE.md

## Что это

CLI-инструмент для анализа целевой аудитории из чатов вебинаров.
На входе — CSV-выгрузка из чата. На выходе — интерактивный HTML-дашборд с JTBD-анализом, портретами ЦА, рекомендациями для оффера.

**Автор:** Дмитрий Ледовских
**Назначение:** анализ аудитории продающих вебинаров для адаптации оффера и контента

---

## Как это работает (полный пайплайн)

```
CSV-выгрузка из чата
        ↓
[Шаг 1] Очистка — убрать мусор, приветствия, модераторов
        ↓
[Шаг 2] JTBD-анализ — категоризация по regex-словарям
        ↓
[Шаг 3] Подсчёт статистики — упоминания, уникальные люди, цитаты
        ↓
[Шаг 4] Генерация портретов ЦА — кластеризация по jobs + профессиям
        ↓
[Шаг 5] Генерация рекомендаций — на основе топ-jobs и страхов
        ↓
[Шаг 6] Сборка HTML-дашборда — интерактивные вкладки с данными
        ↓
HTML-файл готов → деплой на Vercel (опционально)
```

---

## Формат входных данных

CSV с колонками: `Время | id отправителя | Имя | id получателя | Сообщение`

Пример строки:
```
2026-03-09 18:15:32,user123,Алексей Иванов,chat456,Хочу создать бота для записи клиентов в мою парикмахерскую
```

---

## Шаг 1: Очистка чата

**Цель:** убрать мусор, оставить содержательные сообщения.

**Что фильтровать:**
- Сообщения модераторов (по имени)
- Приветствия: «привет», «добрый вечер», «спасибо», «класс», «огонь», «да», «нет», «ок»
- Короткие сообщения < 15 символов (после удаления эмодзи)
- Только эмодзи/символы
- Ссылки
- Спам-повторы символов

**Результат:** очищенный список сообщений с метаданными (время, имя, sender_id)

**Референс:** `reference-scripts/analyze_chat.py`

---

## Шаг 2: JTBD-анализ

**Цель:** каждое сообщение прогнать по 9 категориям Jobs-to-be-Done + профессии + продукты + страхи.

### 9 категорий Jobs (regex-словари в `reference-scripts/jtbd_analysis.py`):

| Job | Ключевые слова (regex) | Формулировка |
|-----|----------------------|--------------|
| ЗАРАБОТАТЬ_ДЕНЬГИ | заработ, деньг, доход, подработ, монетиз | Хочу получить новый источник заработка |
| СОЗДАТЬ_ПРОДУКТ | создать бот, сделать приложени, калькулятор, парсер | Хочу создать продукт без программистов |
| НАЙТИ_КЛИЕНТОВ | найти клиент, привлечь клиент, первый клиент, заявк | Хочу найти клиентов чтобы зарабатывать |
| АВТОМАТИЗИРОВАТЬ | автоматизир, рутин, масштабир, упростить | Хочу автоматизировать процессы |
| СМЕНИТЬ_ПРОФЕССИЮ | уйти из найма, удалённ, новая професси | Хочу работать удалённо и на себя |
| ДОВЕСТИ_ДО_ПРОДАКШНА | не работает, ошибк, хостинг, домен, деплой | Хочу довести прототип до рабочего продукта |
| НАУЧИТЬСЯ_ПРОДАВАТЬ | продавать, сколько брать, ценообразован | Хочу научиться продавать свои продукты |
| РАЗОБРАТЬСЯ_В_ИНСТРУМЕНТАХ | vpn, lovable, claude, какой инструмент | Хочу разобраться что использовать |
| ПОНЯТЬ_НИШУ | определить ниш, не знаю где примени | Хочу определить своё направление |

### 18 категорий профессий:

IT/Digital, Образование, Строительство, Медицина, Маркетплейсы, Сетевой маркетинг, Бьюти, Психология, Юриспруденция, Пенсионеры, Общепит, Копирайтинг, Продажи, Фриланс, Бухгалтерия, Логистика, Безработные, Недвижимость

### Типы желаемых продуктов:

Телеграм-бот, Калькулятор, Парсер, Мобильное приложение, Приложение, Сайт/лендинг, CRM, AI-помощник, AI-агент, Интернет-магазин

### 5 категорий страхов:

- Страх «не получится» (не смогу, боюсь, затык)
- Страх продаж (не умею предлагать, боюсь отказ)
- Дорого / нет денег (платный, бесплатн, инвестировать)
- Техническая беспомощность (сложно, не разбираюсь)
- Блокировки РФ (vpn, недоступен, регион)

### Что считать для каждой категории:

1. **Количество упоминаний** — сколько раз встретилось
2. **Уникальные люди** — сколько разных sender_id (важнее чем упоминания!)
3. **Топ-5 цитат** — дословные цитаты >25 символов, с именем автора

**Референс:** `reference-scripts/jtbd_analysis.py`

---

## Шаг 3: Портреты ЦА

На основе пересечения jobs + профессий — выделить 3 портрета:

**Формат портрета:**
```
Название: «Хочу создать и заработать»
Доля: ~40%
Типичные профессии: наёмный, фрилансер, пенсионер
Jobs: создать продукт, заработать, найти клиентов
Боли: не получается самостоятельно, не знает как продавать
Желания: первые деньги, готовый продукт, работать из дома
Ключевая цитата: дословная из чата
```

---

## Шаг 4: Рекомендации для оффера

На основе данных — сгенерировать 5-7 рекомендаций в формате:

```
Рекомендация: Говори «ЗАРАБОТАТЬ», не «НАУЧИТЬСЯ»
Данные: «Заработать» — Job #1 (118 человек). «Научиться» — даже не в топ-5.
Как применить: В каждом слайде начинай с результата в деньгах, не с процесса.
```

---

## Шаг 5: HTML-дашборд

**Структура дашборда (4 секции, вкладки внутри):**

### Секция 1: День 1
- Портреты ЦА (3 карточки)
- Jobs to be Done (горизонтальные бары, отсортированы по упоминаниям)
- Боли и страхи (бары + ключевые цитаты)
- Желаемые продукты (бары + конкретные примеры с именами)
- Ниши аудитории (бары по уникальным людям)
- Рекомендации для оффера (7 карточек с данными)
- Структура оффера (3 столпа + посылы по портретам)

### Секция 2: День 2 (если есть второй файл)
- Ключевые инсайты (что изменилось)
- Продукты и ниши
- Цитаты дня

### Секция 3: Сравнение дней
- Таблица: день 1 vs день 2 (jobs, страхи, ниши)
- Ключевые сдвиги с рекомендациями

### Секция 4: Возражения ОП (если есть данные от отдела продаж)
- ТОП-10 возражений с частотой
- Цитаты + как закрыть в эфире

**Дизайн:**
- Тёмная тема (фон #0a0a0f)
- Шрифт Inter
- Акценты: фиолетовый (#6c5ce7), зелёный (#00cec9), оранжевый (#e17055)
- Горизонтальные бары для данных
- Карточки для рекомендаций
- Навигация: кнопки секций сверху → подвкладки снизу

**Референс полного дашборда:** `examples/dashboard-example.html`

## Язык аудитории

### Используй дословно:
- «Хочу зарабатывать из дома»
- «Первые клиенты и заявки»
- «То, что можно сразу показать клиенту»
- «Автоматизация, которая экономит часы»
- «Не хочу возвращаться в найм»

### НЕ используй:
- MVP, скейлить, product-market fit
- инновации, трансформация, экосистема
- деплой, рефакторинг, API (без перевода)

---

## Tech Stack

- **Language:** Python 3.12+
- **Зависимости:** только stdlib (csv, re, json, pathlib, collections)
- **Фронтенд:** один HTML-файл (inline CSS + JS, без фреймворков)
- **Деплой:** Vercel (статика, vercel.json с rewrites)

## Что улучшить по сравнению с референсом

1. **Словари в JSON, не в коде** — чтобы менять категории без правки Python
2. **Единый CLI** — одна команда вместо 3 скриптов
3. **HTML-шаблон** — отделить дизайн от данных (Jinja2 или string.Template)
4. **Автопортреты** — кластеризация не вручную, а по пересечению jobs + профессий
5. **Авторекомендации** — генерировать на основе топ-3 jobs и топ-3 страхов
6. **Поддержка нескольких дней** — сравнение автоматически
7. **Vercel-ready** — сразу деплоится как статика
